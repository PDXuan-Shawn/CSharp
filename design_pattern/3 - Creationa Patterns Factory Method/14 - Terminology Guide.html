📜 Terminology Guide

  • 🏭 Factory Method Pattern: A design pattern that provides an interface for creating objects but allows subclasses to alter the type of objects that will be created.

  • 📦 Product:

    • Definition: Represents the object that the factory method creates.

    • In our example: The Payment interface is the product. It represents a general payment method.

  • 🛠️ Concrete Product:

    • Definition: A specific implementation of the product.

    • In our example: CreditCardPayment, PayPalPayment, and BankTransferPayment are concrete products. They define how each specific payment method processes a payment.

  • 🌐 Creator:

    • Definition: An abstract class that declares the factory method.

    • In our example: The PaymentFactory class is the creator. It provides a method to create a payment but doesn't specify which type.

  • 🔨 Concrete Creator:

    • Definition: A subclass that implements the factory method and returns an instance of a concrete product.

    • In our example: CreditCardPaymentFactory, PayPalPaymentFactory, and BankTransferPaymentFactory are concrete creators. Each one knows how to create a specific payment method.

  • 📜 Abstract Method:

    • Definition: A method declared in an abstract class but doesn't have any implementation. Subclasses are expected to provide a concrete implementation for this method.

    • In our example: The process method in the Payment class and the create_payment method in the PaymentFactory class are abstract methods.

  • 📝 Concrete Class:

    • Definition: A class that provides implementations for all of its inherited abstract methods.

    • In our example: CreditCardPayment, PayPalPayment, and BankTransferPayment are concrete classes because they provide implementations for the process method.
