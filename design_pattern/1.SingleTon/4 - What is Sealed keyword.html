What is the sealed keyword in C#? 🤔

In C#, the sealed keyword is used to prevent a class from being inherited by any other class. In simpler terms, if you mark a class as sealed, no other class can inherit from it. It's like putting a lock 🔒 on the class!

Why use sealed? 💡

  • Safety: To ensure that no other classes can change or extend the behavior of your class.

  • Performance: Sometimes, sealing a class can lead to minor performance improvements because the compiler knows that the class won't have any derived classes.

Example 📝:

Let's see it in action:

public sealed class SealedClass
{
    public void ShowMessage()
    {
        Console.WriteLine("Hello from the sealed class!");
    }
}

In this code, SealedClass is a class that is marked as sealed. This means:

🚫 You cannot do this:

public class ChildClass : SealedClass
{
    // This will give a compilation error because you can't inherit from a sealed class.
}

✅ But you can do this:

public class MyClass
{
    SealedClass myObject = new SealedClass();
    myObject.ShowMessage();  // This will print "Hello from the sealed class!"
}

In summary, the sealed keyword in C# is like a "No Trespassing" 🚫 sign for classes. It tells other classes: "You can use me, but you can't extend or change me!".
